## 设计原则

### 1. 问题导向型结构
**为什么选这个**：学术写作的黄金法则——读者跟随发现过程，学得最深

**核心流程**：
```
Post-LN 有梯度问题？
    ↓
Pre-LN 解决了吗？没有，只是反向了
    ↓（梯度流向图展示两种问题）
为什么梯度不均衡很糟糕？（类比+直觉解释）
    ↓
NormFormer 的三处改进如何解决
    ↓
实验验证效果
```

### 2. 梯度流向图的作用
**位置**：引言末尾（第 1.3 或 1.4 节）  
**作用**：直观展示三种架构的梯度分布差异
```
Post-LN：  梯度 ████░░░░░░ （深层爆炸，早层饥荒）
Pre-LN：   梯度 ░░░░████░░ （早层过量，深层饥荒）
NormFormer: 梯度 ░░██████░░ （各层均衡）
```

**格式建议**：用 PNG 图表，宽度 800px，清晰展示层别（x轴）vs 梯度范数（y轴），三条线对比

## 内容优化要点

### 1. 类比与直觉
- **梯度失配类比**：工厂流水线、葡萄酒倒杯子等贴切类比
- **为什么梯度不均很糟**：直接回答读者的"为什么我应该关心"

### 2. 公式简化
- 去掉不必要的中间推导，只保留关键公式
- 用纯文本解释比数学符号优先

### 3. 逻辑流转
- 每个改进点后立即跟消融数据，闭合"这个改进有多重要"的问题
- 避免在讲完所有改进后才集中讲实验

### 4. 视觉分层
- 标题清晰（h2/h3 层级）
- 关键发现用 **粗体** 突出
- 表格和图表前加简短说明（"如表 X 所示"）
