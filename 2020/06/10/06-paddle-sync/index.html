paddle 同步训练源码阅读笔记

distributed system
Notes
Publish Date:   2020-06-10

paddlepadlle 同步CPU分布式

sync_train.py

  1. class PaddleCloudRoleMaker()

    role maker for paddle cloud
    读取系统里面的一系列环境变量，判断当前机器是PSERVER or TRAINER
    获取其他机器的 ip:port 等等
    获取index

  2. def fleet.init(role)
    fleet 本身是个 DistributedTranspiler类
    这个类的主要的作用是

    1. 初始化一个role_maker对象，生成本机里面的role

    2. 进行optimizer的config 和 program的config

    3. 如果本机是server，还能初始化server，控制server的运行

    4. 如果本机是worker，创建worker进程进行训练

      init()主要是将初始化fleet里面的role_maker成员，并且调用role_maker类的generate_role函数，进行本机的role的初始化

  3. class DistributeTranspilerConfig()
    主要为分布式的transpiler jobs提供config的支持
    比如

    1. 同步异步

    2. 是否使用geo_sgd算法

    3. Minimum number of split elements in block

    4. 是否实时切割信息等等（注意实时切割信息和sync为True冲突

      等等

  4. def fleet.distributed_optimizer(optimizer, strategy)
    创建fleet的_optimizer成员
    该成员是一个TranspilerOptimizer()类
    继承自 DistributedOptimizer
    实现了DistributedOptimizer()里面的三个虚函数

    1. backward() loss回传,计算梯度
    2. apply_gradients 更新梯度
    3. minimize() 由上面两个函数组成

  5. optimizer.minimize(avg_cost)
    输入：

    1. loss 产生的loss

    2. scopes 无作用

    3. startup_program 初始化parameter_list参数的program

    4. parameter_list 需要update的参数列表

    5. no_grad_set 不需要更新的参数列表

      输出:

      optimize_ops, 附加的ops的列表
      params_grads (param, grad) 参数及产生的梯度

  6. fleet._transpile(config)
    主要的作用，设置config
    设置main_program 主要的运行program
    设置startup_program init_server init_worker运行的program

  7. is_server() is_worker()
    返回role_maker类创建的本机的信息

  8. init_server() run_server()
    开始跑startup_program、main_program

  9. init_worker()
    首先，等待所有的参数服务器p_server启动
    第二，运行executor初始化start_up program
    第三，等待所有的worker全部初始化

    检查所有的服务器是否正常
    创建一个Communicator()类

    Communicator用于分布式训练的信息传递
    是CPP类Communicator()的封装

  10. exe.run(fleet.startup_program)
    初始化含有分布式流程的fleet.startup_program

  11. exec_strategy build_strategy
    exec_strategy CPP类 core.ParallelExecutor.ExecutionStrategy
    包含一些计算图计算时的参数
    build_strategy CPP类 core.ParallelExecutor.BuildStrategy
    包含一些计算图构造时的参数

  12. fluid.compiler.CompiledProgram(fleet.main_program).with_data_parallel(

    loss_name=avg_cost.name,
build_strategy=build_strategy,
exec_strategy=exec_strategy

    )

    class CompiledProgram()
    根据build_strategy的配置，CompiledProgram用于转换program或计算图以进行各种优化。
    例如，运算符在计算图中的融合，在计算图中执行过程中的内存优化等。

    with_data_parallel()
    该接口用于将输入的程序或图转换为 multi-graph，以在数据并行模式下运行模型。 用户可以使用build_strategy和exec_strategy来设置一些可在Graph的构建和计算过程中应用的优化，例如减少AllReduce操作的数量，指定运行模型的计算Graph中使用的线程池的大小，以此类推。

    输入:

    1. loss_name 回传的loss
    2. build_strategy 策略类
    3. exec_strategy 策略类
    4. share_vars_from CompiledProgram()类，和设定的CompiledProgram进行参数共享
    5. places 指定模型运行在哪些设备上面cpu gpu

    输出：

    CompiledProgram() 自身的对象

  13. fleet.is_first_worker()
    判断是不是第一个worker节点，这个由role_maker里面的信息得到

  14. fleet.stop_worker()
    self._communicator.stop() 关闭communicator
    self._role_maker._finalize() 析构role_maker
    self._executor.close() 关闭executor
